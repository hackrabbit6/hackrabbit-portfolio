# hackrabbit 作品集重设计 · 概念 A

## 0. 背景与问题

现有页面是一份"干净 SaaS 风"的规格表(技能列表 / 项目列表 / 经历列表),
结构与文案都是 AI 一键可批量生成的样式 —— 这正是用户要逃离的"廉价批量 AI 感"。

**真正的卡点不是审美或技术,是"永远不满意"的无限循环**:用户反复推翻重做那已经
OK 的 80%,却没收掉扎眼的 20%,且每次只是换一个同样没有"自己"的模板。

**根因诊断**:网站里没有"这个人的脑子"。身份层里有一批独一无二、几乎没人这么定位
自己的素材 —— hackrabbit 黑客兔身份、第一性原理 + 贝叶斯思维 OS、"未来最值钱的是
管理 AI 员工的人"的论点、健康/现金流/认知/AI 四支柱 —— 一个字都没出现在网站上。

**解法**:不重设计、不换模板,而是把"这个人的思想与身份"灌进去,并用一个被确定
下来的核心概念(概念 A)给出边界,以关掉无限循环。

## 1. 核心概念(锁定)

**概念 A — AI 智能体操盘手 × 黑客兔。**

把用户最稀缺、别人最难抄的人设摆到台前:一个用第一性原理思考、指挥一队 AI 智能体
交付产品的前端工程师;视觉用 hackrabbit 黑客兔身份包装。

> 选定即边界。之后任何改动只问"这符不符合概念 A?",不再问"够不够好看"。

## 2. 灵魂(锁定)

- **人设句(hero 主标题)**:
  "**4-5 年前端交付** + 我用**第一性原理**思考、指挥一队 **AI 智能体**,把活干得更快更稳。"
  > 站位:信誉打底 + operator 加分。遵守在案定位决策
  > (`~/Ideaverse/codex/Projects/hackrabbit-portfolio/decisions/2026-06-30-concept-a-credibility-first.md`)——
  > 先立 4-5 年业务前端交付可信度,AI operator 是差异化加分,**不否定写码、不显得放弃稳定前端**。
- **声音**:全站第一人称、有观点。把第三人称规格表文案("负责…/参与…")改成
  "我看到 X,所以我这么做"。文案要带"我"的判断 —— 判断是 AI 抄不出来的。
- **黑客兔 mark**:自画字形/符号(参考 `(\(\` `(•ᴥ•)` `/ づ`),全站一致复用。

## 3. 视觉身份 · Design Token(锁死,不准凭感觉调)

氛围:**深色电光 · 黑客控制台**(冷、精密、黑客感,但克制 —— 不是 gamer RGB)。

- 背景 `#0B0B0E`
- 文字:火白 / 灰白(主文案灰白,强调火白)
- **主强调 `#39FF8B`(琴荧绿)** + 电青作副强调
- 字体:等宽字体承载 hacker / operator 身份 + 一款有性格的标题字
- 间距阶梯:`4 / 8 / 12 / 16 / 24 / 40 / 64`
- 一套固定字号阶梯
- 全部落地为 CSS 变量,全站不允许散落的硬编码色值 / 间距值

## 4. 页面架构(在现有结构上重组,不推翻)

| 区块 | 干什么 | 改动 |
|---|---|---|
| Hero | 一句人设 + 黑客兔 mark + 四支柱微缩(健康/现金流/认知/AI) + `> crew: N agents online` | 改 |
| **🐇 My AI Crew** | **全站王牌(新增)**:展示如何指挥 agent —— OpenSpec → Claude Code / Codex → 交付的真实工作流,记忆系统截图。没有任何前端会这样展示自己。 | 新增 |
| Selected Projects | 每个项目改成"人 × agent 协作":我定方向(第一性原理拆解)→ agent 执行 → 我验收。**删掉空假窗口装饰(`project-visual` 里的空 `visual-lines`)**。 | 改 |
| Skill Map | 保留,每项加一行"我怎么用 AI 放大它" | 微调 |
| 经历 / 联系 | 基本保留,文案转第一人称 | 微调 |

### 4.1 项目卡片改写模式(统一模板)

每个项目卡按同一结构叙事,替换现有的规格罗列:
1. **方向**:我用第一性原理怎么拆这个问题 / 为什么做
2. **协作**:我与哪些 agent 怎么分工执行
3. **结果**:交付了什么、可演示 / 可投递的产出

并删除顶部的空假浏览器窗口装饰 —— 它是当前唯一把页面拉到"半成品/模板"档的元素。

## 5. ✅ 完成清单(治"永远不满意"的药 —— 勾完即上线,不许再改)

- [ ] Hero 人设句 + 黑客兔 mark 上线
- [ ] My AI Crew 区块有 ≥3 个真实 agent 工作流
- [ ] 4 个项目全部转成"人 × agent"叙事,空假窗口已删
- [ ] Design token 落地为 CSS 变量,全站无散值
- [ ] 全站文案第一人称,无"负责/参与"式规格表句
- [ ] `bun run build` 通过 + `bun run lint` 通过
- [ ] 桌面 + 移动两端截图自检无硬伤
- [ ] **以上勾完 → 强制 deploy,不再开新一轮**

"完成"由这张清单客观定义,不由"感觉对不对"定义。这是本设计治本的关键产物。

## 6. 不做(YAGNI · 防循环)

- ❌ 不推翻现有结构重写
- ❌ 不加博客 / CMS / 动效炫技
- ❌ 不做主题切换(深色控制台是唯一主题)
- ❌ 勾完清单后不许"再调一版"

## 7. 技术约束

- 沿用现有栈:React 19 + TypeScript + Vite + Bun;`lucide-react` 图标。
- `three.js` / `ThreeBackdrop`:Hero 乱线要么收敛成"有意图"的版本(呼应 agent/节点
  网络),要么删除换纯净深色留白 —— 实现阶段二选一,不再无限调。
- 部署:Vercel(Vite preset,`bun run build` → `dist`)。
